## Step 1 — Fill in your product requirements

Open `docs/PRD.md` and describe:
- What is this product?
- Who is it for?
- What problem does it solve?

This is the most important document. Agents will read it at the start of every feature.

---

## Step 2 — Create your first User Story

```bash
cp docs/templates/HU-template.md docs/tasks/HU-001-your-feature.md
```

Fill in the `title`, acceptance criteria, and user story. Leave `flowforge_slug: ""` — it will be filled automatically when you run `/flow-start`.

---

## Step 3 — Install FlowForge IDE packs (if not already done)

```bash
# From the FlowForge repo:
bash /path/to/FlowForge/ide/install.sh __PROJECT_PATH__
```

This installs Cursor rules, OpenCode/Kilo bundles, VS Code agents (Copilot + Kilo), and the Antigravity workflows and rules.

After this step the project contains the IDE directories:

```
.github/agents/          ← GitHub Copilot agents
.opencode/agents/        ← OpenCode + Kilo agents
.kilo/agents/            ← Duplicate of `.opencode/agents/` for Kilo Code
.cursor/                 ← Cursor rules, agents, commands
.agents/rules/           ← Antigravity rules (workflow, git-sin-push, etc.)
.agents/workflows/       ← Antigravity `/flow-*` workflows
```

---

## Step 4 — Start your first feature

In your IDE:

```
/flow-start HU-001-your-feature
```

FlowForge will:
1. Run discovery (search memory + existing code patterns)
2. Write `spec.md` based on your HU
3. Ask you to approve (CKP-1)
4. Then write `plan.md`
5. Ask you to approve (CKP-2)
6. Then write code

## Adoption levels

You are currently at **L1** (docs only). You can start using FlowForge agents immediately (L2) — just run `/flow-start`.

| Level | What you get |
|-------|-------------|
| **L1** | `docs/` structure, HU backlog — no agents required |
| **L2** | Full FlowForge workflow (`/flow-start` → `/flow-close`) |
| **L3** | Add `docs/flowdoc-ciclo.md` for team sprint rhythm |
| **L4** | Add metrics via `forge-memory/metrics` skill |
